Sample Code for Combining Assays with Xars
on  LabKey Server  updated for v 8.2

 

Author:  Peter Hussey

            LabKey Software

            peter@labkey.com

 

Updated:  June 6, 2008

 

The Assay facility, documented as part of the Study application, provides a user-interface over the definition and collection of metadata associated with an experiment run.  As of version 8.2, Assays have not been tied to MS2 experiment runs and protocols in the main source code of LabKey Server.  This sample code hooks up Assays with MS2 runs.  Functionally, it replaces the “Describe MS2 Runs” step in the steps for searching for peptides.  The sample code installs two new Assay providers:  a generic “XarAssay” provider and a specific subclass “Cptac” assay provider.  The Cptac provider is designed for use in the CPTAC initiative with pre-defined metadata properties.

 

Installation

Unzip the distribution files into a temporary directory.  The compiled module is called xarassay.module  and should copied into a directory called “externalModules” under the LabKey installation root.  If you are installing on Windows and have used the setup program with the path defaults, the installation root is

 

C:\Program Files\LabKey Server

 

Create a directory named externalModules under this root and copy xarassay.module into this directory.  The stop and restart the LabKey Tomcat server.

Describing a run

Therer are a few optional paths to the Cptac assay:

  • You can describe runs of the MyCptacAssay  in the project folder itself or in any subfolder. This example uses a subfolder. 
  • There are two entry points into the dialog for associating an Assay with an mzXML data file.:  from the Data Pipeline web part button “Process and Upload Data”, or from the “Upload Runs” button of the Assay List details page for an assay.  This example uses the Data Pipeline button   
  • The default Cptac Assay assumes that each file may have come from a different starting sample and asks for run properties for every file in the directory.  This example uses the default.  An optional Fractionation xar template describes all mzxml files in a directory as belonging to fractions of the same sample. 

 

The following steps show the choices indicated.

  1. In the folder, select the Process  and Upload Data button in the Data Pipeline part.  In the resulting Pipeline browser dialog, open the Peroxisomal_ICAT directory containing 3 mzXML files. 
  2. Press the Create Assay Run button
  3. You should get a Data Upload page generated from the Assay definition.  Select the appropriate values in the Upload Set dropdowns.
  4. Press Next, the  Run Properties section of the form is shown.
  5. Fill out a name and select the starting sample for the file that is listed as the current file in the Run Data dialog box.  The Comments and other fields can also be filled in.
  6. Press Save and Upload Another Run.  After a few seconds you should get the message “Upload Successful” at the top, and a new Current file listed.
  7. Repeat steps 6 and 7 until there is one more file left, then press Save and Finish instead of Save and Upload Another Run. 
  8. In the resulting runs grid, click on the name of a run to see a graphical representation of its steps.
  9. To initiate scoring of the same mzXML files,
    1. select the Process  and Upload Data button in the Data Pipeline part
    2. this time select the X!Tandem Peptide Search button instead of the Xar Assay button.
  10. As per the CPAS Demo tutorial, select the existing yeast_ICAT search protocol and press Search.
